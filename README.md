# 🛒 Store Silpo System (C++)

<img width="1024" height="682" alt="magaz" src="https://github.com/user-attachments/assets/10fe26f4-bef3-476b-b31d-990b4be387a8" />

Консольна система керування замовленнями для магазину, розроблена на мові C++. Проєкт дозволяє створювати замовлення, додавати товари з каталогу або вручну, застосовувати промокоди та зберігати результати у форматі JSON.

## 🚀 Основні можливості
* **Керування товарами:** додавання (з файлу або вручну), видалення та зміна кількості.
* **Гнучка система замовлень:** встановлення дати, статусу (Очікування, Обробка, Доставлено) та методу оплати.
* **Система знижок:** підтримка промокодів для автоматичного перерахунку суми.
* **Робота з даними:**
  * Читання каталогу товарів з `products.txt`.
  * Збереження та завантаження замовлень у форматі `JSON` (використовується бібліотека `nlohmann/json`).
* **Звукове сповіщення:** система видає сигнал при успішному завершенні оплати.

## 🛠 Технології
* **Мова:** C++
* **Фреймворк тестування:** MS Native Unit Test Framework
* **Бібліотеки:**
  * `nlohmann/json` для роботи з даними.
  * `fstream` для работы с файлами.
  * `windows.h` для системних звуків.

## 🧪 Юніт-тестування
Для забезпечення стабільності та коректності роботи ключових модулів проєкту розроблено набір автоматичних тестів. Тести перевіряють роботу базових класів та бізнес-логіки розрахунку вартості замовлень.

### Що покривають тести:
* **Валідація сутностей (Product):** перевірка коректності роботи конструктора за замовчуванням та конструктора з параметрами.
* **Управління кошиком (Order):** додавання унікальних товарів, збільшення кількості вже наявних позицій, видалення товарів з перерахунком суми та зміна кількості одиниць.
* **Система промокодів:** валідація знижок (20%, 10%) та поведінка системи за умови введення неіснуючого коду.
* **Серіалізація даних:** перевірка збереження стану замовлення у файл `JSON` та його подальшого коректного зчитування.

### ⚠️ Демонстрація роботи тестів (Помилка логіки)
З метою демонстрації процесу налагодження програми та перевірки реакції тест-раннера на баги, один із тестів свідомо працює **неправильно**.

У тесті `TestPromoSAVE10` навмисно викликається промокод `"SAVE20"` замість `"SAVE10"`. Оскільки очікуване значення (`90.0`) не збігається з фактично розрахованим системою (`80.0`), цей тест **падає** (стає червоним), що наочно ілюструє роботу автоматичного контролю якості коду.

## 📦 Структура проєкту
* `product.h / product.cpp` — клас товару (ID, назва, ціна, категорія).
* `order.h / order.cpp` — логіка замовлення, розрахунок Total, робота з JSON.
* `main.cpp` — консольне меню користувача.
* `UnitTest1/` — проєкт з юніт-тестами для перевірки логіки програми.
